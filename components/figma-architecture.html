05 The 5 Differentiating Sections What makes this theme impossible to copy from a generic one
Material Callout
⭐ Unique · PDP + Homepage
No theme gives material its own design section. Off-white bg, gold-warm left border, 3-col grid. Converts browsers into buyers.
  • Metal name + finish (Sterling Silver, Oxidized)
  • Weight indication (Light / Medium / Substantial)
  • Sourcing note (Recycled / Local supplier)
  • Schema: fully customizable per product
Maker Story Block
⭐ Unique · Homepage + About
Built around the individual maker — not a generic about section. Portrait + italic Playfair Display name + Mulish craft description.
  • Maker portrait (square crop)
  • Name in italic Playfair Display
  • Pronouns field (optional schema)
  • Studio city + location tag
Process Strip
⭐ Unique · Homepage + About
3–5 images in horizontal sequence. Step number in italic Playfair Display gold-warm, Mulish 200 caption. Communicates handmade immediately.
  • 3–5 configurable steps
  • Square crop — consistent rhythm
  • Step number: italic Playfair Display
  • Horizontal scroll on mobile
Small Catalog Grid
⭐ Unique · Collection Page
Standard grids feel empty at 20 products. 3-col layout with approved card design — curated, not sparse. Filter bar included.
  • 3-col desktop, 2-col tablet, 1-col mobile
  • Hover: CTA slide-up, 0.28s cubic-bezier
  • Terracotta low-stock pill
  • Filter: material / ring size / price
Care + Sizing Guide
⭐ Unique · PDP
Sizing anxiety is the #1 conversion blocker for jewelry. First-class accordion design on PDP — not a PDF link.
  • Ring sizing chart (inline)
  • Metal care guide per material
  • Storage tips
  • Accordion — collapsed by default
Standard Sections
Required · Shopify Compliance
All Shopify-required sections included and reskinned to match the design language: graphite palette, Mulish body, Playfair headings.
  • Header (transparent → solid on scroll)
  • Footer (graphite bg, 3-column)
  • Announcement bar (graphite + gold dot)
  • Newsletter (single-field, minimal)

07 Figma Build Order Follow this sequence — don't jump to pages before system is solid
Phase 1 — Design System (Build First)
  • Set up Variables: Primitives collection (16 hex values from Section 01)
  • Set up Semantic collection (all token aliases from Section 01)
  • Create text styles: 8 type scales from Section 02
  • Create effect styles: border-radius 2px · no shadows
  • Build button set: 5 variants × 3 sizes × default/hover/disabled
  • Build product card set: 5 states (default, hover, sold-out, low-stock, new)
  • Build variant selectors: chip + swatch, all states
  • Build tags + badges: all 7 variants
  • Build inputs: default, focus (graphite border), error (ember), disabled
  • Build utility: announcement bar, breadcrumb, stepper, 3 toast types, gold divider
  • Build nav header: 4 states
  • Build cart drawer: 3 states
Phase 2 — Core Pages (In This Order)
  • PDP — gallery 1.18fr/1fr, details 50/50, material panel, care accordions, trust grid
  • Collection — 3-col grid, filter bar, sort controls
  • Homepage — hero → process → featured → catalog → maker story
  • About / Maker — maker story block + process strip
  • Cart page — full page + drawer component
  • Search — results grid + empty state + predictive overlay
  • Blog / Journal — editorial article layout
  • Supporting: Contact, 404, All Collections
  • Mobile: all pages at 390px after desktop approved
  • Prototype: nav → PDP → cart → checkout flow
